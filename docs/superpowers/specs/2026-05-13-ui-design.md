# op_client UI Design Spec
_2026-05-13_

## Summary

Custom UI design system for op_client — a KMP app targeting Android and Linux Desktop. No Material Design. Built entirely on Compose Multiplatform primitives. Design language: minimal, functional, content-first. Book cover art is the dominant visual element; chrome stays quiet.

---

## 1. Design Direction

**Paradigm:** Minimal / Functional
- Interface recedes; content (covers, titles, authors) takes focus
- No decorative elements, no gradients in chrome, no shadows on surfaces
- Structure communicated through spacing and thin borders, not elevation
- Jost's geometric character sets the visual personality across all text

## 3. Typography

**Font family:** [Jost](https://fonts.google.com/specimen/Jost) — bundled as app resource in both Android and Desktop modules. Same binary, consistent rendering across platforms.

| Role | Weight | Size | Letter-spacing | Transform |
|------|--------|------|---------------|-----------|
| Screen title | 300 | 14sp | +3px | Uppercase |
| Section label | 300 | 9sp | +2.5px | Uppercase |
| Nav label | 300–400 | 7–9sp | +1.5px | Uppercase |
| Book title | 500 | 11–14sp | +0.3px | None |
| Author / metadata | 300 | 9–11sp | +1.5–2px | Uppercase |
| Tag / chip | 400 | 8–9sp | +1px | Uppercase |
| Body / description | 300–400 | 12sp | 0 | None |
| Button | 500 | 9sp | +2px | Uppercase |

Key principle: **spaced uppercase for all UI labels** (nav, section headers, tags, buttons). Jost light at wide tracking is the signature visual gesture of this design system.

---

## 4. Shape

**Radius:** 4px on all interactive surfaces (cards, inputs, buttons, tags, sidebar icons).

**Book covers:** 2px radius only — covers are photographic/illustrated; a harder edge reads as intentional framing.

**No shadows.** Surfaces are distinguished by background color and a 1px `colorBorder` outline, not elevation.

---

## 5. Navigation

Navigation is platform-specific. Shared ViewModels hold destination state; screen shells are in `androidMain` / `jvmMain`.

### Android
- **Bottom navigation bar** — 3 destinations: Search, Browse, Library
- Bar background: `colorSurface`, top border: `colorBorder` (1px)
- Active item: icon + label in `colorAccent`
- Inactive item: icon + label in `colorTextSecondary`
- Label: Jost 300, 7sp, +1.5px, uppercase

### Linux Desktop
- **Collapsed icon sidebar** — 3 destinations: Search, Browse, Library
- Sidebar width: 56dp, background: `colorSurface`, right border: `colorBorder` (1px)
- Icons: 28×28dp, 4px radius, `colorSurface2` background; active: `colorAccentLight`
- Active icon lines/dots: `colorAccent`; inactive: `colorTextSecondary`
- No text labels — icons only (tooltips on hover)
- Top bar: `colorSurface`, height 52dp, contains search input + contextual actions

### Shared destination set
| Destination | Description |
|-------------|-------------|
| Search | Default home screen; search input + results |
| Browse | Subject taxonomy and category navigation |
| Library | Personal lists and reading log |

## 6. Component Inventory

Components live in `commonMain` as Compose Multiplatform composables unless marked platform-specific.

### Inputs
- `SearchInput` — full-width, `colorSurface` fill, `colorBorder` outline, 4px radius, Jost 300 placeholder in uppercase+tracked
- `FilterChip` — `colorAccentLight` bg, `colorAccent` text, 4px radius, toggleable

### Lists
- `BookRow` — cover thumbnail (34×48dp mobile / 42×60dp desktop) + title + author + tag row; `colorBorder` bottom divider; selected state: `colorSurface2` background
- `SectionLabel` — Jost 300, uppercase, tracked, `colorTextSecondary`

### Cards
- `BookCard` — grid layout for Browse screen; cover image dominant; title + author below in Jost
- `DetailPanel` — Desktop only (`jvmMain`): right-hand panel showing cover, metadata stats, primary action button

### Actions
- `PrimaryButton` — `colorAccent` fill, white text, 4px radius, Jost 500 uppercase tracked
- `SecondaryButton` — transparent fill, `colorBorder` outline, `colorTextSecondary` text
- `IconButton` — `colorSurface2` bg, 4px radius, 28×28dp (sidebar) or 40×40dp touch target

### Tags
- `SubjectTag` — `colorAccentLight` bg, `colorAccent` text, 4px radius, Jost 400 8–9sp uppercase

### Feedback
- `LoadingState` — centered minimal spinner in `colorAccent`; no skeleton screens
- `EmptyState` — centered Jost 300 label in `colorTextSecondary`; no illustrations
- `ErrorState` — inline error text in `colorTextSecondary`; retry link in `colorAccent`

## 7. Layout

### Android
- `contentPadding`: 14dp horizontal, 10dp vertical
- `listItemPadding`: 9dp vertical, 0dp horizontal (full-width dividers)
- `sectionLabelMargin`: 10dp top, 8dp bottom

### Desktop
- `contentPadding`: 20dp
- `sidebarWidth`: 56dp
- `topBarHeight`: 52dp
- `detailPanelWidth`: 220dp (shown when a result is selected)
- Max content width: unconstrained — layout fills window

### Responsive (Desktop)
- Window width < 800dp: hide detail panel, navigate to detail screen on tap (dp = density-independent pixels, same unit Compose Desktop uses)
- Window width ≥ 800dp: master-detail layout (list + detail panel side by side)

## 8. Platform-Specific Adaptations

| Concern | Android | Linux Desktop |
|---------|---------|---------------|
| Navigation chrome | Bottom nav bar | Icon sidebar |
| Detail view | Full-screen push | Side panel (≥800dp) |
| Search input | Top of screen, always visible | In top bar |
| Keyboard nav | Touch-only | Full keyboard support (Tab, arrow keys, Enter) |
| Hover states | None | Subtle `colorSurface2` bg on hovered rows |
| Window management | Activity lifecycle | Compose Desktop `Window`, state-restored on relaunch |
| Context menu | Long-press bottom sheet | Right-click menu |

---

## 9. What This System Excludes

- **Material Design** — no `MaterialTheme`, no `Scaffold`, no Material components
- **Elevation / shadows** — no `Card` elevation; depth via color only
- **Ripple effects** — replace with instant state change (selected bg color)
- **FABs** — no floating action buttons; actions live in top bar or detail panel
- **Snackbars** — inline error/success states only
- **Bottom sheets** — drawer/sheet patterns not used; context menus on Desktop, full-screen dialogs on Android

---

## 10. Implementation Notes

- All color tokens defined as `CompositionLocal` values, resolved at root with `isSystemInDarkTheme()`
- Typography tokens defined as a `TextStyle` object tree — no dependency on `MaterialTheme.typography`
- `AppTheme` composable in `commonMain` provides colors + typography; platform entry points wrap content in `AppTheme`
- Jost font loaded via `FontFamily` with bundled `.ttf` resources in `commonMain/resources`
- No `MaterialTheme` wrapper anywhere in the tree
